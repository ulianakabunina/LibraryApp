# Library Management System

## Описание проекта

Консольное Java-приложение для управления личной библиотекой с интеграцией Google Books API.
Система позволяет добавлять книги вручную или по ISBN, искать книги, изменять статус чтения и просматривать аналитику.

---

# 1. Анализ ситуации и контекста

## Проблема

При ведении личной библиотеки пользователи часто сталкиваются со следующими проблемами:

* отсутствие единого учета книг;
* сложность поиска нужной книги;
* потеря информации о прочитанном;
* отсутствие статистики и аналитики чтения;
* ручной ввод информации о книгах.

## Актуальность проблемы

Неэффективное хранение информации о книгах приводит к:

* потере времени;
* дублированию записей;
* снижению удобства управления библиотекой;
* отсутствию контроля прогресса чтения.

## Контекст использования

Система предназначена для:

* студентов;
* преподавателей;
* любителей чтения;
* пользователей, ведущих личную библиотеку.

Приложение автоматизирует работу с книгами за счет подключения к Google Books API.

## Схема текущего процесса

```text
Пользователь
    ↓
Ручной поиск информации о книге
    ↓
Ручное заполнение данных
    ↓
Хранение записей
```

| ----------------- | ---------------------- | ------------------------- |
| Excel             | Простота использования | Нет автоматизации         |

## Выбор подхода

Для проекта был выбран подход консольного Java-приложения, так как он:

* прост в реализации;
* не требует сложной инфраструктуры;
* позволяет изучить ООП, API и коллекции Java;
* обеспечивает быструю работу с данными.

# 3. Концепция решения

## Основная идея

Система хранит информацию о книгах и автоматизирует их добавление через Google Books API.

## Возможности системы

* добавление книги вручную;
* добавление книги по ISBN;
* поиск книг;
* удаление книг;
* изменение статуса чтения;
* хранение оценок и рецензий;
* аналитика библиотеки.

## Принцип работы

```text
Пользователь
    ↓
Console UI
    ↓
LibraryManager
    ↓
Google Books API
```

## Основные компоненты

| Компонент      | Назначение    |
| -------------- | ------------- |
| Main           | Главное меню  |
| LibraryManager | Бизнес-логика |
| Book           | Модель книги  |

## Преимущества решения

* автоматическое получение данных о книге;
* простая архитектура;
* удобный поиск;
* поддержка аналитики;
* использование современных Java-технологий.

# 4. Архитектура решения и детализация внедрения

## Архитектура приложения

```text
Main
  |
  v
LibraryManager
  |
  v
Book
```

## Алгоритм добавления книги по ISBN

```text
Ввод ISBN
   ↓
Запрос к Google Books API
   ↓
Получение JSON-ответа
   ↓
Извлечение данных
   ↓
Создание объекта Book
   ↓
Добавление в библиотеку
```

## Используемые технологии

* Java 17
* Collections Framework
* Stream API
* HttpClient
* Google Books API

## Структура данных

### Класс Book

Содержит:

* ISBN;
* название;
* автора;
* жанр;
* формат;
* статус;
* рейтинг;
* даты чтения.

### Класс LibraryManager

Отвечает за:

* хранение книг;
* поиск;
* фильтрацию;
* аналитику;
* работу с API.

### Класс Main

Отвечает за:

* меню;
* взаимодействие с пользователем;
* обработку команд.
