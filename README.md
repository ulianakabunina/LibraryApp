# Система управления личной библиотекой

## Описание проекта
Консольное приложение на Java для каталогизации, анализа и сохранения личных библиотек с поддержкой бумажных, электронных книг и аудиокниг. Интеграция с Google Books API для автоматического получения метаданных по ISBN.

---

## 1. Анализ ситуации и контекста

### Ключевые неэффективности текущего процесса

| Проблема | Текущее состояние | Влияние |
|----------|------------------|----------|
| Разрозненность данных | Бумажные книги в одном месте, электронные в другом, аудиокниги не учитываются | Потеря информации о 25% книг |
| Ручной учет | Записи в блокнотах, Excel, заметках телефона | 30% времени уходит на поиск книг |
| Отсутствие системы | Нет напоминаний о начатых книгах, нет трекинга чтения | 15% книг покупаются повторно |
| Нет аналитики | Невозможно понять предпочтения по жанрам и авторам | Невозможно оптимизировать пополнение библиотеки |

### Влияние проблемы

**Количественные потери:**
- 25% книг теряются из виду (куплены, но не прочитаны)
- 30% времени читателя уходит на поиск нужной книги
- 15% книг покупаются повторно из-за отсутствия учета
- 10% записей дублируются в разных файлах

**Качественные риски:**
- Снижение мотивации к чтению из-за хаотичной организации
- Потеря времени на рутинные операции учета
- Невозможность отследить прогресс и планы чтения

### Контекст проблемы

Проблема возникает в следующих сценариях:
1. **Личные библиотеки** книголюбов (100-1000+ книг)
2. **Исследовательские коллекции** (80% бумажные, 15% электронные, 5% аудио)
3. **Семейные библиотеки** с разными форматами изданий

**Факторы, влияющие на существование проблемы:**
- Отсутствие единого стандарта учета
- Человеческий фактор (лень записывать, забывчивость)
- Разнообразие форматов книг
- Отсутствие автоматизированных инструментов для обычного пользователя

## 2. Обзор существующих решений

### Сравнительный анализ аналогов

| Решение | Плюсы | Минусы | Цена | Поддержка ISBN |
|---------|-------|--------|------|----------------|
| **Excel/Google Sheets** | Гибкость, доступность | Ручной ввод, нет аналитики, нет поиска | Бесплатно | Нет |
| **Блокнот/Заметки** | Простота | Нет структуры, потеря данных | Бесплатно | Нет |
| **Calibre** | Мощный для электронных книг | Сложный, нет бумажных книг, тяжелый | Бесплатно | Да |
| **Goodreads** | Социальная сеть, аналитика | Зависимость от интернета, реклама | Бесплатно | Да |
| **Libib** | Хороший каталог | Ограничения бесплатной версии | Freemium | Да |
| **Наше решение** | Все форматы, оффлайн, API интеграция | Консоль (нет GUI) | Бесплатно | Да |

### SWOT-анализ существующих решений
СИЛЬНЫЕ СТОРОНЫ (Strengths):
├── Бесплатные аналоги широко доступны
├── Calibre имеет мощный функционал для EPUB
├── Goodreads имеет социальные функции
└── Excel гибок для продвинутых пользователей

СЛАБЫЕ СТОРОНЫ (Weaknesses):
├── Нет единого решения для всех форматов
├── Ручной ввод данных (3-5 минут на книгу)
├── Нет аналитики по потерянным книгам
├── Сложность поиска (5-15 минут)
└── Неудобство работы со статусами чтения

ВОЗМОЖНОСТИ (Opportunities):
├── Автоматизация через API Google Books
├── Создание единой базы всех форматов
├── Аналитика предпочтений пользователя
├── Интеграция с другими сервисами
└── Мобильная версия в будущем

УГРОЗЫ (Threats):
├── Google Books API может измениться
├── Появление новых конкурентов
├── Требуется подключение к интернету для API
└── Сложность миграции из других систем

text

### Ключевые тренды в управлении личными библиотеками

1. **API-интеграция** - автоматическое получение метаданных
2. **Кросс-форматность** - единый каталог для всех типов книг
3. **Аналитика** - визуализация читательских привычек
4. **Оффлайн-доступ** - работа без интернета (кроме API)
5. **Простота использования** - минимальный порог входа

### Обоснование выбора подхода

**Почему консольное приложение на Java?**
- Кроссплатформенность (Windows, Mac, Linux)
- Высокая производительность
- Простота распространения (один JAR файл)
- Минимальные системные требования
- Отсутствие зависимости от браузера и интернета (кроме API)

**Почему Google Books API?**
- Бесплатный (до 1000 запросов/день)
- Широкое покрытие книг
- Богатые метаданные (обложки, страницы, жанры)
- Стабильный и документированный
